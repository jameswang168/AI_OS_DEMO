## 一、项目愿景与核心理念

**"天予 Local AI"** 是一套安装在**移动 SSD** 上的**私有 AI 系统**。它的终极目标是：

1. **物理级隐私保护** — 敏感数据绝不出港，在本地熔断；
2. **硬件解耦** — 插上任何宿主机（从 i7-4790 到高端服务器）即可点火运行；
3. **智力套利** — 借助免费/低价云端大模型算力，但通过脱敏代号化让云端只做"失忆型工具人"；
4. **反向造血** — 利用纠错数据集向 AI 厂商变现。

### 三方协同模式
| 角色 | 职责 |
|------|------|
| **云端架构师** (网页端 AI) | 提供算法逻辑、框架伪代码、高级策略规划 |
| **本地实施者** (Antigravity / 本地 AI) | 落实代码实现、文件操作、本地测试验证 |
| **指挥官** (人类用户) | 调配两边信息，审阅逻辑，给出人工评分与方向控制 |

## 四、核心架构详解

### 4.1 分层防御体系 — L1 / L2 / L3

系统采用**三层漏斗式过滤**架构：

```
   用户输入
      │
      ▼
 ┌─────────────────┐
 │   L1 宏观过滤    │  filter_l1.py
 │  · Markdown 语义分块 (按标题层级)
 │  · sensitive_gate 装饰器: 敏感词打标 (privacy_level 0/1)
 │  · 内存倒排索引 (KeywordIndexer) + 本地序列化缓存
 └────────┬────────┘
          │ Chunk[]  (带 privacy_level + content_hash)
          ▼
 ┌─────────────────┐
 │   L2 精炼嵌入    │  embedder_l2.py
 │  · 防线1: privacy_level==1 → 抛 SecurityViolationError 熔断
 │  · 防线2: SHA-256 哈希缓存命中 → 直接读本地向量
 │  · 防线3: 调用 Provider 接口生成向量 → 写入缓存
 └────────┬────────┘
          │ Vector[]
          ▼
 ┌─────────────────┐
 │   L3 混合路由    │  api_gate/router.py
 │  · Mode A (🔒 全本地): 一切不外发，仅用本地索引
 │  · Mode B (🌐 混合动力): Level 0 允许云端, Level 1 强制本地
 └─────────────────┘
```

### 4.2 Provider 抽象工厂模式

```python
# backend/core/providers/base.py
class BaseEmbeddingProvider(ABC):
    @abstractmethod
    def embed_text(self, text: str) -> List[float]: ...
```

- 目前已实现: `GeminiProvider` (基于 `urllib.request`，零依赖)
- **关键预留位**: `ollama_provider.py` — 当 Tesla P100 显卡上线时，只需实现此类即可无缝切换至全本地离线推理

### 4.3 硬件旁路感知 (Hardware Bypass)

`hardware_bypass.py` 负责扫描宿主机硬件能力：
- **High_Power**: 检测到本地 GPU → 全本地精炼
- **Stealth_Bypass**: 无 GPU → Scout 并发 + L1/L2 持久拦截

目前为**占位模拟**状态，逻辑框架已就绪，待 P100 硬件接入时激活。

## 五、Phase B 插件生态详解

Phase B 是系统的"商业化变现层"，采用插件架构，全部注册在 `plugins/manifest.json` 中：

### 5.1 Scout Hub — 侦察兵控制中心
- **功能**: 用 `ThreadPoolExecutor` 并发向多个云端 API 发送"逻辑陷阱"探针
- **输出**: `data/leaderboard.json` (综合延时 + 智力分排行榜)
- **选出"首席工蜂"**: 当日最佳性价比云端 API

### 5.2 Colosseum Judge — 斗兽场博弈器
- **功能**: 同一问题发给两个模型分身，第三方裁判打分
- **输出**: `logs/gladiator_pairs.json` (DPO chosen/rejected 纠错对)

### 5.3 Refinery Mint — 数据精炼厂
- **功能**: 将人类反馈 (`logs/human_feedback.json`) + 对撞对 → DPO `.jsonl` 格式
- **安全**: 内置**指纹擦除器** (Fingerprint Erase)，移除路径/IP 等隐私残留
- **输出**: `data/refinery_export/dpo_train_pairs.jsonl`

### 5.4 Report Anonymizer — 匿名化转换器
- **功能**: 将 `leaderboard.json` 去除厂商名称，用 Node_A/B/C 掩饰
- **输出**: `data/sales/anonymous_report.md` (《2026 私有 AI 逻辑稳定性蓝皮书》)

### 5.5 Agent Diplomat — 外交官
- **功能**: 生成向厂商发送的"兼容性邀请"非侵略性邮件模板
- **分模板**: 优等生 (表现好) vs 差生 (有幻觉) 差异化话术

### 5.6 Agent Treasurer — 影子财务官
- **功能**: 生成一次性阅后即焚收款地址 (模拟 HD Wallet)
- **核心机制**: AI 只能**准备交易负载**，无法触碰私钥；必须经过指挥官**物理点击**才能签名发布
- **数据**: `vault/pending_tx/` (挂起) → `vault/shadow_ledger.json` (已签名)

### 5.7 Commerce Suite — 商贸编排器
- **功能**: 将上述插件串联为完整**商业化流水线**：
  `Scout 发现幻觉 → Anonymizer 脱标识 → Diplomat 拟函 → Treasurer 挂账`

### 5.8 Close Phase B — Phase B 结算
- **功能**: 汇聚 DPO 数据集数量、匿名评级数、影子账单余额
- **输出**: `data/Phase_B_Final_Assets.json` (智力资产储量报告)

## 十二、关键设计决策记录

| 决策 | 原因 |
|------|------|
| **100% 标准库, 零 pip install** | 目标是插入老旧电脑即用，不依赖网络安装 |
| **SHA-256 哈希缓存而非数据库** | 文件级持久化，断电 SSD 拔出后再插入可瞬间恢复 |
| **装饰器模式实现 sensitive_gate** | 解耦隐私逻辑与业务逻辑，任何 parse 方法加 `@sensitive_gate` 即自动拦截 |
| **Provider 抽象工厂** | 为 P100 本地推理 (Ollama) 预留无缝切换接口 |
| **分离"准备交易"与"物理签名"** | AI 无权私自发起任何资金操作，必须人类物理确认 |
| **90/10 法则** | 90% 通用外围交云端加速，10% 核心算法锁死本地（Crypto Accelerator） |
